00 Kod Kapsama Analizi Nedir?

Kod kapsama (code coverage), testlerin kaynak kodun ne kadarını çalıştırdığını ölçen bir yazılım kalite metriğidir.

Testler çalışırken hangi kod satırlarının, dallarının ve fonksiyonlarının işlendiği izlenir. Sonuç yüzde olarak ifade edilir. Yüksek kapsama, testlerin kodun geniş bir kesimini doğruladığını gösterir — ancak %100 kapsama, hata olmadığı anlamına gelmez.

Kapsama türleri

Line coverage
Her kaynak satırının en az bir kez çalıştırılıp çalıştırılmadığı. En basit ve en yaygın metrik. gcov varsayılan olarak bunu ölçer.
Branch coverage
if/else, switch, ternary gibi dallanma noktalarının her iki kolunun da test edilip edilmediği. Line coverage'dan daha güçlü bir garanti.
Function coverage
Her fonksiyonun en az bir kez çağrılıp çağrılmadığı. Dead code tespitinde yararlıdır.
MC/DC
Modified Condition/Decision Coverage — havacılık ve tıbbi yazılım için zorunlu (DO-178C, IEC 61508). gcov bu seviyeyi desteklemez.

EMBEDDED'DE KAPSAMA KULLANIMI

Kod kapsama genellikle cross-compile edilen kodun host (x86) üzerinde birim testi ile ölçülür. Board üzerinde kapsama verisi toplamak mümkündür ama .gcda dosyalarını host'a kopyalamak gerekir. Kernel kapsama için ayrı bir mekanizma (debugfs) kullanılır.

01 Userspace gcov — --coverage Flag

GCC'nin --coverage flag'i, kapsama ölçümü için gerekli enstrümantasyonu otomatik ekler.

Derleme ve çalıştırma

bash — gcov derleme adımları●
# ─── Derleme — --coverage = -fprofile-arcs + -ftest-coverage ──
gcc --coverage -O0 -g -o mylib_test mylib.c test_mylib.c

# Derleme sonrası oluşan dosyalar:
# mylib.gcno — kapsama grafiği (derleme zamanı)
# test_mylib.gcno

# ─── Çalıştır — .gcda dosyaları oluşur ──────────────────────
./mylib_test

# Çalıştırma sonrası oluşan dosyalar:
# mylib.gcda — çalıştırma verisi
# test_mylib.gcda

# ─── gcov raporu üret ────────────────────────────────────────
gcov mylib.c

# Çıktı: mylib.c.gcov (annotated source)
# -:    0:Source:mylib.c
# -:    1:#include "mylib.h"
# 5:   10:int mymath_add(int a, int b) { return a + b; }
# -:   11:
# #####:   14:int mymath_div(int a, int b) { ... }
# Anlamı: 5=5 kez çalıştı, #####=hiç çalışmadı

CMake ile --coverage entegrasyonu

CMakeLists.txt — coverage hedefi●
cmake_minimum_required(VERSION 3.16)
project(mylib C)

option(ENABLE_COVERAGE "Kod kapsama analizi etkinleştir" OFF)

if(ENABLE_COVERAGE)
    if(NOT CMAKE_BUILD_TYPE STREQUAL "Debug")
        message(WARNING "Coverage için Debug build önerilir")
    endif()
    add_compile_options(--coverage -O0 -g)
    add_link_options(--coverage)
endif()

add_library(mylib mylib.c)
add_executable(test_mylib tests/test_mylib.c)
target_link_libraries(test_mylib mylib)

# Özel coverage hedefi
find_program(LCOV lcov)
find_program(GENHTML genhtml)

if(LCOV AND GENHTML)
    add_custom_target(coverage
        COMMENT "Kapsama raporu oluşturuluyor..."
        COMMAND ${CMAKE_CTEST_COMMAND} --output-on-failure
        COMMAND ${LCOV}
            --capture
            --directory ${CMAKE_BINARY_DIR}
            --output-file coverage.info
        COMMAND ${LCOV}
            --remove coverage.info
            "/usr/*" "tests/*"
            --output-file coverage-filtered.info
        COMMAND ${GENHTML}
            coverage-filtered.info
            --output-directory coverage-html
        WORKING_DIRECTORY ${CMAKE_BINARY_DIR}
        DEPENDS test_mylib
    )
endif()

Cross-compile uyarısı

bash — cross-compile senaryosu●
# Cross-compile ile gcov kullanımı — DİKKAT:
# --coverage ile cross-compile edilen binary board'da çalışır
# .gcda dosyaları board'da oluşur, host'a kopyalanmalıdır

# Adım 1: Cross-compile
aarch64-poky-linux-gcc --coverage -O0 -g \
  -o test_mylib_arm mylib.c tests/test_mylib.c \
  --sysroot=/opt/poky-sdk/sysroots/aarch64-poky-linux

# Adım 2: Board'a kopyala ve çalıştır
scp test_mylib_arm root@192.168.1.100:/tmp/
ssh root@192.168.1.100 "/tmp/test_mylib_arm"

# Adım 3: .gcda dosyalarını host'a kopyala
scp root@192.168.1.100:/tmp/*.gcda .

# Adım 4: gcov'u doğru araç ile çalıştır
aarch64-poky-linux-gcov mylib.c

# GCOV_PREFIX ile .gcda kayıt dizini kontrol edilebilir:
ssh root@192.168.1.100 "GCOV_PREFIX=/tmp/gcov GCOV_PREFIX_STRIP=4 /tmp/test_mylib_arm"

02 lcov Kurulumu ve Kullanımı

lcov, gcov'un çıktısını toplar, filtreler ve genhtml için hazır hale getirir. Birden fazla test çalıştırmasını birleştirme yeteneği de sunar.

lcov kurulumu

bash — lcov kurulumu●
# Ubuntu/Debian
apt install -y lcov

# Sürüm kontrolü (1.16+ önerilir)
lcov --version

# Kaynaktan derleme (son sürüm için)
git clone https://github.com/linux-test-project/lcov.git
make -C lcov install PREFIX=/usr/local

lcov iş akışı

bash — lcov tam iş akışı●
# ─── 1. Başlangıç sayaçlarını sıfırla ────────────────────────
lcov --zerocounters --directory .

# ─── 2. Başlangıç (boş) kapsama al — coverage için gerekli ──
lcov --capture --initial \
  --directory . \
  --output-file coverage-baseline.info

# ─── 3. Testleri çalıştır ────────────────────────────────────
./build/test_mylib
./build/test_mylib_integration

# ─── 4. Kapsama verisini topla ───────────────────────────────
lcov --capture \
  --directory . \
  --output-file coverage-tests.info

# ─── 5. Baseline ile birleştir ───────────────────────────────
lcov --add-tracefile coverage-baseline.info \
  --add-tracefile coverage-tests.info \
  --output-file coverage-combined.info

# ─── 6. Sistem dosyaları ve test kodunu hariç tut ───────────
lcov --remove coverage-combined.info \
  "/usr/*" \
  "*/tests/*" \
  "*/build/*" \
  --output-file coverage-filtered.info

# ─── 7. Özet çıktı ──────────────────────────────────────────
lcov --summary coverage-filtered.info

# Çıktı örneği:
# Reading tracefile coverage-filtered.info
# Summary coverage rate:
#   lines......: 87.3% (312 of 357 lines)
#   functions..: 92.1% (47 of 51 functions)
#   branches...: 74.6% (125 of 168 branches)

Birden fazla test run'ını birleştirme

bash — lcov --add-tracefile●
# Test suite 1
./build/test_unit
lcov --capture --directory . --output-file run1.info

# Test suite 2
./build/test_integration
lcov --capture --directory . --output-file run2.info

# Test suite 3 (board'dan gelen .gcda ile)
scp root@board:/tmp/*.gcda build/src/
lcov --capture --directory . --output-file run3.info

# Tüm run'ları birleştir
lcov \
  --add-tracefile run1.info \
  --add-tracefile run2.info \
  --add-tracefile run3.info \
  --output-file coverage-total.info

03 HTML Rapor — genhtml

genhtml, lcov'un .info dosyasından etkileşimli HTML kapsama raporu oluşturur.

genhtml kullanımı

bash — genhtml rapor oluşturma●

genhtml coverage-filtered.info \
  --output-directory coverage-html \
  --title "mylib Kapsama Raporu" \

  --output-directory coverage-html \
  --branch-coverage \

cd coverage-html && python3 -m http.server 8080
# Tarayıcıda: http://localhost:8080

04 Kernel gcov

Linux kernel, gcov enstrümantasyonunu destekler. debugfs aracılığıyla çalışma zamanında kapsama verisi toplanabilir.

Kernel konfigürasyonu

kernel .config — gcov etkinleştirme●
# menuconfig: Kernel hacking → GCOV-based kernel profiling
CONFIG_GCOV_KERNEL=y
CONFIG_GCOV_FORMAT_AUTODETECT=y

# Belirli bir modül/subsystem için
# Makefile'a ekle:
# GCOV_PROFILE_mydriver.o := y
# veya tüm dizin için:
# GCOV_PROFILE := y

# debugfs gerekli:
CONFIG_DEBUG_FS=y

Kapsama verisi toplama

bash — kernel gcov veri toplama●
# debugfs mount (genellikle otomatik):
mount -t debugfs none /sys/kernel/debug

# gcov dizini
ls /sys/kernel/debug/gcov/
# Çıktı: kernel kaynak ağacı yapısında .gcda dosyaları

# Kapsama sayaçlarını sıfırla (test öncesi)
echo 1 > /sys/kernel/debug/gcov/reset

# Testleri çalıştır (kernel kodu tetikle)
insmod /lib/modules/$(uname -r)/extra/mydriver.ko
/usr/bin/mydriver-test
rmmod mydriver

# .gcda dosyalarını host'a kopyala
KERNEL_SRC=/usr/src/linux
scp -r root@board:/sys/kernel/debug/gcov/ ./kernel-gcov/

# Host'ta gcov raporunu oluştur (kernel source tree ile)
lcov --capture \
  --directory kernel-gcov/sys/kernel/debug/gcov/ \
  --gcov-tool gcov \
  --output-file kernel-coverage.info

genhtml kernel-coverage.info \
  --output-directory kernel-coverage-html \
  --title "Kernel gcov Coverage"

gcov-tool ile veri birleştirme

bash — gcov-tool kullanımı●
# Birden fazla test run'ından gelen .gcda dosyalarını birleştir
gcov-tool merge \
  -o merged-gcda/ \
  run1-gcda/ \
  run2-gcda/

# Kapsama sıfırlama (belirli dosya için)
gcov-tool rewrite \
  --scale 0 \
  mydriver.gcda \
  -o mydriver-reset.gcda
KERNEL GCOV UYARISI

Kernel gcov, bellek ve performans açısından maliyetlidir. Production sistemlerde kesinlikle devre dışı bırakılmalıdır. Kernel kapsama testleri için özel bir test image/konfigürasyonu kullanın. Yocto'da: KERNEL_FEATURES:append = " features/gcov/gcov.scc"

05 CI Kapsama Kapısı

Kapsama kapısı (coverage gate), belirlenen eşiğin altına düşen merge request'leri otomatik olarak reddeder.

GitLab CI — kapsama kapısı

.gitlab-ci.yml — kapsama kapısı●
coverage:
  stage: test
  image: gcc:13
  before_script:
    - apt-get update && apt-get install -y lcov python3-pip cmake
    - pip3 install gcovr
  script:
    # Derleme
    - cmake -B build -DENABLE_COVERAGE=ON -DCMAKE_BUILD_TYPE=Debug
    - cmake --build build --parallel
    # Test + kapsama topla
    - cd build && ctest --output-on-failure
    # lcov ile bilgi topla
    - lcov --capture --directory . --output-file coverage.info
    - lcov --remove coverage.info "/usr/*" "*/tests/*" --output-file coverage-filtered.info
    - genhtml coverage-filtered.info --output-directory coverage-html --branch-coverage
    # Cobertura XML (GitLab test coverage görselleştirmesi)
    - gcovr --root .. --exclude "tests/.*" --xml-pretty --output ../coverage.xml
    # Kapsama kapısı — %80 altında başarısız ol
    - python3 ../ci/check-coverage.py --min-line 80 --min-branch 70 coverage-filtered.info
  # GitLab'a kapsama yüzdesini bildir (regex ile parse)
  coverage: '/lines\.*: (\d+\.\d+)\%/'
  artifacts:
    when: always
    paths:
      - coverage-html/
      - coverage.xml
    reports:
      coverage_report:
        coverage_format: cobertura
        path: coverage.xml

check-coverage.py — kapsama kapısı scripti

ci/check-coverage.py●
#!/usr/bin/env python3
import argparse, re, sys

def parse_lcov_summary(info_file):
    lines_hit = lines_total = 0
    branches_hit = branches_total = 0
    with open(info_file) as f:
        for line in f:
            if line.startswith("LH:"):  lines_hit    += int(line[3:])
            if line.startswith("LF:"):  lines_total  += int(line[3:])
            if line.startswith("BRH:"): branches_hit   += int(line[4:])
            if line.startswith("BRF:"): branches_total += int(line[4:])
    return {
        "line":   (lines_hit    / lines_total    * 100) if lines_total    else 0,
        "branch": (branches_hit / branches_total * 100) if branches_total else 0,
    }

def main():
    ap = argparse.ArgumentParser()
    ap.add_argument("info_file")
    ap.add_argument("--min-line",   type=float, default=80.0)
    ap.add_argument("--min-branch", type=float, default=0.0)
    args = ap.parse_args()

    cov = parse_lcov_summary(args.info_file)
    print(f"Satır kapsama:  {cov['line']:.1f}% (min: {args.min_line}%)")
    print(f"Dal kapsama:    {cov['branch']:.1f}% (min: {args.min_branch}%)")

    failed = False
    if cov["line"] < args.min_line:
        print(f"HATA: Satır kapsama {cov['line']:.1f}% < {args.min_line}%", file=sys.stderr)
        failed = True
    if cov["branch"] < args.min_branch:
        print(f"HATA: Dal kapsama {cov['branch']:.1f}% < {args.min_branch}%", file=sys.stderr)
        failed = True

    sys.exit(1 if failed else 0)

if __name__ == "__main__": main()

06 Exclude Patterns

Test kodu, otomatik oluşturulan kod ve ilgisiz sistem dosyaları kapsama raporundan hariç tutulmalıdır.

Kaynak kodda exclusion marker'lar

C — LCOV_EXCL marker'ları●
/* Tek satır hariç tut */
assert(ptr != NULL); /* LCOV_EXCL_LINE */

/* Blok hariç tut */
/* LCOV_EXCL_START */
static void debug_print(const char *msg) {
    fprintf(stderr, "[DEBUG] %s\n", msg);
}
/* LCOV_EXCL_STOP */

/* Belirli bir dal hariç tut */
if (unlikely(error)) { /* LCOV_EXCL_BR_LINE */
    handle_fatal_error();
}

lcov --remove ile pattern exclude

bash — lcov exclude patterns●
lcov --remove coverage.info \
  # Sistem başlıkları
  "/usr/*" \
  "/opt/poky-sdk/*" \
  # Test kodu
  "*/tests/*" \
  "*/test_*.c" \
  "*_test.c" \
  # Otomatik oluşturulan dosyalar
  "*/build/*" \
  "*/generated/*" \
  "*/proto/*" \
  # Mock dosyaları
  "*/mock_*.c" \
  "*/stub_*.c" \
  --output-file coverage-filtered.info

gcovr ile exclude

bash — gcovr exclude patterns●
gcovr \
  --root . \
  # Regex pattern'lar (Python re formatı)
  --exclude "tests/.*" \
  --exclude ".*/build/.*" \
  --exclude ".*/generated/.*" \
  --exclude ".*_test\.c$" \
  --exclude ".*mock.*" \
  # Belirli bir fonksiyonu hariç tut
  --exclude-lines "LCOV_EXCL_LINE" \
  --exclude-branches-by-pattern "assert\(.*\)" \
  --xml-pretty \
  --output coverage.xml

07 Pratik: %80 Kapsama Hedefi + CI Pipeline

Embedded C kütüphanesi için %80 satır kapsama hedefi olan tam CI pipeline'ı.

Proje yapısı

  mylib/
  ├── CMakeLists.txt
  ├── .lcovrc
  ├── src/
  │   ├── mylib.c
  │   ├── mylib_crypto.c
  │   └── mylib_parser.c
  ├── include/
  │   └── mylib.h
  ├── tests/
  │   ├── CMakeLists.txt
  │   ├── test_mylib.c
  │   ├── test_crypto.c
  │   └── test_parser.c
  └── ci/
      ├── check-coverage.py
      └── run-coverage.sh

ci/run-coverage.sh

ci/run-coverage.sh●
#!/bin/bash
set -euo pipefail

BUILD_DIR="build-coverage"
MIN_LINE_COV=${1:-80}   # İlk argüman veya varsayılan %80
MIN_BRANCH_COV=${2:-70}

echo "=== Kapsama build: min_line=${MIN_LINE_COV}% min_branch=${MIN_BRANCH_COV}% ==="

# CMake konfigürasyonu
cmake -B ${BUILD_DIR} \
  -DENABLE_COVERAGE=ON \
  -DCMAKE_BUILD_TYPE=Debug \
  -DCMAKE_C_COMPILER=gcc \
  -G Ninja

# Derleme
cmake --build ${BUILD_DIR} --parallel $(nproc)

# Sayaçları sıfırla
lcov --zerocounters --directory ${BUILD_DIR}

# Testleri çalıştır
cmake --build ${BUILD_DIR} --target test -- ARGS="--output-on-failure"

# Kapsama topla
lcov --capture \
  --directory ${BUILD_DIR} \
  --output-file coverage.info \
  --quiet

# Filtrele
lcov --remove coverage.info \
  "/usr/*" "*/tests/*" "*/build-coverage/*" \
  --output-file coverage-filtered.info \
  --quiet

# HTML rapor oluştur
genhtml coverage-filtered.info \
  --output-directory coverage-html \
  --branch-coverage \
  --title "mylib Coverage" \
  --quiet

# Cobertura XML
gcovr --root . \
  --exclude "tests/.*" --exclude "build-coverage/.*" \
  --xml-pretty --output coverage.xml

# Özet göster
lcov --summary coverage-filtered.info

# Kapsama kapısı kontrolü
python3 ci/check-coverage.py \
  --min-line ${MIN_LINE_COV} \
  --min-branch ${MIN_BRANCH_COV} \
  coverage-filtered.info

echo "=== HTML rapor: coverage-html/index.html ==="
