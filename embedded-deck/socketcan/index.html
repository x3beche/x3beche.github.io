00 CAN bus nedir

CAN (Controller Area Network), 1983'te Bosch tarafından otomotiv elektroniği için geliştirilen, diferansiyel sinyalleme kullanan çok-master seri iletişim protokolüdür.

Tarihçe ve motivasyon

1980'lerin başında modern otomobiller onlarca ECU (Electronic Control Unit) barındırmaya başlamıştı. Her ECU arasında ayrı kablo döşemek hem ağır hem de güvenilmezdi. BMW 8-Serisi'nde 1980'lerde 2 km kablo bulunuyordu. Bosch'un 1983'te geliştirdiği CAN, tüm bu kontrol birimlerinin tek bir diferansiyel çift üzerinden haberleşmesini sağladı. 1991'den itibaren otomotiv ağ standardı olarak benimsendi ve bugün endüstriyel otomasyon, tıbbi cihazlar, uçak aviyoniği ve robotikten uzanan bir yelpazeye yayıldı.

Diferansiyel sinyalleme ve gürültü bağışıklığı

CAN bus iki kablo kullanır: CAN_H (High) ve CAN_L (Low). Sinyal, bu iki kablo arasındaki voltaj farkından okunur. Elektromanyetik gürültü her iki kabloya eşit biçimde etki ettiğinden fark değişmez; bu sayede motor kompartımanı gibi gürültülü ortamlarda hata oranı dramatik biçimde düşer.

  CAN_H  ──────────────────── 2.5V (recessive) / 3.5V (dominant)
  CAN_L  ──────────────────── 2.5V (recessive) / 1.5V (dominant)

  Dominant bit: CAN_H - CAN_L = 2V
  Recessive bit: CAN_H - CAN_L ≈ 0V

  Bus uçlarında 120Ω sonlandırma direnci gereklidir.

CAN ID ve arbitration

CAN bus'ta multi-master arbitration bit-wise AND ile çalışır. Birden fazla node aynı anda iletim başlatırsa, düşük ID her zaman kazanır (dominant bit kazanır). Bu sayede ekstra handshake olmadan öncelik belirlenir. Standard frame 11-bit ID (2048 farklı mesaj), Extended frame 29-bit ID (537 milyon farklı mesaj) kullanır.

NOT

CAN bus'ta herhangi bir node hata tespit ettiğinde Error Frame göndererek tüm node'ları uyarır. Sürekli hata gönderen node önce "Error Passive", ardından "Bus-Off" durumuna geçer ve bus'tan kendini ayırır. Linux SocketCAN bu durumları CAN_ERR_MASK ile kullanıcı alanına raporlar.

01 SocketCAN mimarisi

Linux'ta CAN bus, ağ arayüzü olarak modellenir: ip link ile yönetilir, standart socket API'siyle veri alınıp gönderilir.

Linux'ta CAN modeli

SocketCAN'in temel tasarım kararı, CAN bus'ı bir ağ arayüzü olarak modellemektir. Bu sayede can0, can1 gibi interface'ler eth0 gibi görünür; ip link set can0 up ile etkinleştirilir. Uygulama geliştirici ise AF_CAN socket family'sini kullanarak standart write() / read() çağrılarıyla CAN frame gönderip alır.

  Kullanıcı Alanı
  ┌─────────────────────────────────────────┐
  │  Application (socket API)               │
  │  socket(PF_CAN, SOCK_RAW, CAN_RAW)      │
  └──────────────┬──────────────────────────┘
                 │ AF_CAN
  ───────────────│──────── Kernel Alanı ────
  ┌──────────────▼──────────────────────────┐
  │  SocketCAN subsystem                    │
  │  (net/can/)                             │
  │  ├─ can_raw    (raw frame)              │
  │  ├─ can_bcm    (broadcast manager)      │
  │  └─ can_isotp  (ISO-TP segmentation)    │
  └──────────────┬──────────────────────────┘
                 │ netdev
  ┌──────────────▼──────────────────────────┐
  │  CAN Network Driver                     │
  │  (drivers/net/can/)                     │
  │  ├─ vcan  (virtual, loopback)           │
  │  ├─ mcp251x  (SPI)                      │
  │  ├─ flexcan  (i.MX, NXP)               │
  │  └─ peak_usb, kvaser_usb, ...           │
  └─────────────────────────────────────────┘

02 Virtual CAN (vcan) ile loopback test

Gerçek donanım olmadan CAN geliştirmesi yapmak için vcan sürücüsü bir loopback interface oluşturur — aynı arayüzde gönderilen frame'ler anında alınır.

vcan arayüzü kurulumu

vcan-setup.sh●

sudo modprobe vcan

# vcan0 arayüzü oluştur

sudo ip link set vcan0 up

# Doğrula
ip link show vcan0
# → 4: vcan0: <NOARP,UP,LOWER_UP> ...

# Birden fazla virtual interface (test ortamı)
sudo ip link add dev vcan1 type vcan

İlk test: candump + cansend

İki terminal açın. Birinde dinleyin, diğerinde gönderin:

terminal 1 — dinle●

# Bekler — frame gelince yazdırır
terminal 2 — gönder●

cansend vcan0 123#DEADBEEF

# Extended 29-bit ID (büyük ID değeri)

# terminal 1 çıktısı:
#   vcan0  123   [4]  DE AD BE EF
#   vcan0  12345678   [8]  01 02 03 04 05 06 07 08

CAN frame formatı

  cansend formatı:
  ─────────────────────────────────────────────────────────
  <interface>  <ID>#<DATA>

  Standard (11-bit):   123#DEADBEEF
  Extended (29-bit):   12345678#DEADBEEF   (8 hex digit ID)
  RTR frame:           7DF#R8              (R + DLC)
  CAN FD:              123##0DEADBEEF      (## separator)

  DATA: 0–16 byte (FD), her byte 2 hex digit, büyük/küçük harf fark etmez

cangen ile otomatik frame üretimi

terminal●
# Her 10 ms'de bir rastgele frame üret, 8 byte, rastgele ID
cangen vcan0 -g 10 -I i -L 8

# Sabit ID, rastgele veri, her 5 ms
cangen vcan0 -g 5 -I 0x123 -L 8

# Sonsuz döngü yerine N frame üret
cangen vcan0 -n 100 -g 10
NOT

vcan arayüzü sistemin her yeniden başlatılmasında sıfırlanır. Otomatik kurulum için /etc/network/interfaces veya bir systemd servisi oluşturun. Yocto tabanlı sistemlerde bunu recipe içinde do_install() ile yapılandırabilirsiniz.

Bu bölümde

  • modprobe vcan + ip link add/set ile sanal CAN arayüzü kurulumu
  • candump ve cansend temel kullanımı
  • CAN frame formatı: standard, extended, RTR, CAN FD sözdizimi
  • cangen ile yük testi ve random frame üretimi

05 SocketCAN C API

CAN uygulaması yazmak için standart POSIX socket API kullanılır: AF_CAN, struct can_frame ve write/read sistematik çağrıları ile doğrudan CAN frame alışverişi yapılır.

Gerekli başlık dosyaları ve yapılar

can_types.h (referans)●
#include <linux/can.h>
#include <linux/can/raw.h>
#include <net/if.h>
#include <sys/ioctl.h>
#include <sys/socket.h>

/* Temel CAN frame yapısı (linux/can.h'den) */
struct can_frame {
    canid_t  can_id;     /* 32-bit: ID + EFF/RTR/ERR bayrakları */
    __u8     can_dlc;   /* Veri uzunluğu (0–8)                */
    __u8     __pad;
    __u8     __res0;
    __u8     __res1;
    __u8     data[8] __attribute__((aligned(8)));
};

/* ID bayrak bitleri */
/* CAN_EFF_FLAG (0x80000000): Extended 29-bit ID */
/* CAN_RTR_FLAG (0x40000000): Remote Transmission Request */
/* CAN_ERR_FLAG (0x20000000): Error frame */

Tam C örneği: producer + consumer

can_producer.c●
#include <stdio.h>
#include <string.h>
#include <unistd.h>
#include <net/if.h>
#include <sys/ioctl.h>
#include <sys/socket.h>
#include <linux/can.h>
#include <linux/can/raw.h>

int can_open(const char *ifname) {
    int sock;
    struct sockaddr_can addr;
    struct ifreq        ifr;

    /* CAN raw socket oluştur */
    sock = socket(PF_CAN, SOCK_RAW, CAN_RAW);
    if (sock < 0) { perror("socket"); return -1; }

    /* Interface indeksini al */
    strncpy(ifr.ifr_name, ifname, IFNAMSIZ);
    if (ioctl(sock, SIOCGIFINDEX, &ifr) < 0) {
        perror("ioctl SIOCGIFINDEX"); return -1;
    }

    /* Socket'i CAN arayüzüne bağla */
    memset(&addr, 0, sizeof(addr));
    addr.can_family  = AF_CAN;
    addr.can_ifindex = ifr.ifr_ifindex;

    if (bind(sock, (struct sockaddr *)&addr, sizeof(addr)) < 0) {
        perror("bind"); return -1;
    }
    return sock;
}

int main(void) {
    int sock = can_open("vcan0");
    if (sock < 0) return 1;

    struct can_frame frame;
    memset(&frame, 0, sizeof(frame));

    /* Standard 11-bit ID: 0x123 */
    frame.can_id  = 0x123;
    frame.can_dlc = 4;
    frame.data[0] = 0xDE;
    frame.data[1] = 0xAD;
    frame.data[2] = 0xBE;
    frame.data[3] = 0xEF;

    ssize_t nbytes = write(sock, &frame, sizeof(frame));
    if (nbytes != sizeof(frame))
        perror("write");
    else
        printf("Frame gönderildi: ID=0x%03X DLC=%d\n",
               frame.can_id, frame.can_dlc);

    close(sock);
    return 0;
}
can_consumer.c●
/* can_open() aynı — sadece receive kısmı gösteriliyor */

void can_receive_loop(int sock) {
    struct can_frame frame;

    /* CAN filter kur: yalnızca 0x100–0x1FF ID aralığını al */
    struct can_filter rfilter[1];
    rfilter[0].can_id   = 0x100;
    rfilter[0].can_mask = CAN_SFF_MASK & 0x700;  /* üst 4 bit mask */
    setsockopt(sock, SOL_CAN_RAW, CAN_RAW_FILTER,
               &rfilter, sizeof(rfilter));

    /* Error frame bildirimini aktif et */
    can_err_mask_t err_mask = CAN_ERR_MASK;
    setsockopt(sock, SOL_CAN_RAW, CAN_RAW_ERR_FILTER,
               &err_mask, sizeof(err_mask));

    while (1) {
        ssize_t nbytes = read(sock, &frame, sizeof(frame));
        if (nbytes < 0) { perror("read"); break; }

        /* Error frame tespiti */
        if (frame.can_id & CAN_ERR_FLAG) {
            fprintf(stderr, "CAN error: 0x%08X\n", frame.can_id);
            continue;
        }

        printf("ID=0x%03X DLC=%d data:", frame.can_id, frame.can_dlc);
        for (int i = 0; i < frame.can_dlc; i++)
            printf(" %02X", frame.data[i]);
        printf("\n");
    }
}

Bu bölümde

  • socket(PF_CAN, SOCK_RAW, CAN_RAW) ile CAN socket oluşturma
  • struct can_frame yapısı: can_id, can_dlc, data[8]
  • ioctl SIOCGIFINDEX + bind ile interface bağlama
  • CAN filter (can_filter + setsockopt) ve error frame bildirim

06 ISO-TP (ISO 15765-2)

CAN frame başına maksimum 8 byte veri sığar; ISO-TP bu sınırı aşmak için CAN frame'leri segmentlere böler ve büyük mesajların gönderilmesini sağlar.

Neden ISO-TP?

OBD-II diagnostik mesajları, ECU yazılım güncellemeleri (UDS, ISO 14229) ve büyük veri transferleri 8 byte'ı aşar. ISO-TP (ISO 15765-2) bu problemi çözmek için bir transport katmanı tanımlar. Maximum 4095 byte'lık mesajlar gönderilebilir.

  Mesaj: 20 byte
  ─────────────────────────────────────────────────────────
  First Frame (FF):     [FF][len_high][len_low][data 0..5]   → 6 byte veri
  Consecutive Frame 1:  [CF 21][data 6..12]                  → 7 byte veri
  Consecutive Frame 2:  [CF 22][data 13..19]                 → 7 byte veri

  Alıcı, FF gelince Flow Control Frame gönderir:
  Flow Control (FC):    [FC 00][blocksize][stmin]
                         │       │          │
                         │       │          └── frame arası min süre (ms)
                         │       └────────────── kaç CF sonra FC beklenir (0=sonsuz)
                         └────────────────────── 00=devam et, 01=bekle, 02=çıkış

ISO-TP socket kullanımı

isotp_example.c●
#include <stdio.h>
#include <string.h>
#include <unistd.h>
#include <net/if.h>
#include <sys/ioctl.h>
#include <sys/socket.h>
#include <linux/can.h>
#include <linux/can/isotp.h>

int isotp_open(const char *ifname,
               canid_t tx_id, canid_t rx_id)
{
    int sock;
    struct sockaddr_can addr;
    struct ifreq        ifr;

    /* SOCK_DGRAM + CAN_ISOTP: mesaj-oriented, segmentation otomatik */
    sock = socket(PF_CAN, SOCK_DGRAM, CAN_ISOTP);
    if (sock < 0) { perror("socket"); return -1; }

    strncpy(ifr.ifr_name, ifname, IFNAMSIZ);
    ioctl(sock, SIOCGIFINDEX, &ifr);

    memset(&addr, 0, sizeof(addr));
    addr.can_family             = AF_CAN;
    addr.can_ifindex            = ifr.ifr_ifindex;
    /* TX ID: bu socket'in gönderdiği frame'lerin CAN ID'si */
    addr.can_addr.tp.tx_id      = tx_id;
    /* RX ID: dinlenen CAN ID */
    addr.can_addr.tp.rx_id      = rx_id;

    if (bind(sock, (struct sockaddr *)&addr, sizeof(addr)) < 0) {
        perror("bind"); return -1;
    }
    return sock;
}

int main(void) {
    /* OBD-II: tester 0x7DF'ye gönderir, ECU 0x7E8'den cevap verir */
    int sock = isotp_open("vcan0", 0x7DF, 0x7E8);
    if (sock < 0) return 1;

    /* OBD-II: Mode 01 PID 0D = araç hızı */
    unsigned char req[] = { 0x02, 0x01, 0x0D };
    write(sock, req, sizeof(req));

    unsigned char resp[4096];
    ssize_t n = read(sock, resp, sizeof(resp));
    if (n > 0) {
        printf("Yanıt (%zd byte):", n);
        for (int i = 0; i < n; i++)
            printf(" %02X", resp[i]);
        printf("\n");
    }

    close(sock);
    return 0;
}

bus = can.Bus(interface='socketcan', channel='vcan0')

# Tester (0x7DF) → ECU (0x7E8) bağlantısı
addr = isotp.Address(
    isotp.AddressingMode.Normal_11bits,
    txid=0x7DF,
    rxid=0x7E8
)
layer = isotp.CanStack(bus=bus, address=addr)

# OBD-II Mode 01 PID 0D isteği gönder
layer.send(bytes([0x02, 0x01, 0x0D]))

# Cevabı bekle
import time
for _ in range(100):
    layer.process()
    if layer.available():
        response = layer.recv()
        print(f"Yanıt: {response.hex()}")
        break
    time.sleep(0.01)

bus.shutdown()

Notifier + Listener: asenkron mesaj işleme

can_monitor.py●
import can
import time

class MotorStatusListener(can.Listener):
    """Belirli CAN ID'leri için özel işleyici."""

    def on_message_received(self, msg):
        if msg.arbitration_id == 0x200:
            # Motor hız mesajı: byte 0-1 = RPM (little-endian)
            rpm = int.from_bytes(msg.data[0:2], byteorder='little')
            print(f"Motor RPM: {rpm}")

        elif msg.arbitration_id == 0x201:
            # Sıcaklık mesajı: byte 0 = °C (signed)
            temp = int.from_bytes([msg.data[0]], signed=True)
            print(f"Motor Sıcaklık: {temp}°C")

    def on_error(self, exc):
        print(f"CAN hatası: {exc}")


bus = can.Bus(interface='socketcan', channel='vcan0')

# Listener'ları kaydet
motor_listener   = MotorStatusListener()
printer_listener = can.Printer()                      # terminal çıktısı
logger_listener  = can.Logger("can_log.asc")          # ASC formatında log

# Notifier: bus üzerinden gelen mesajları tüm listener'lara dağıtır
notifier = can.Notifier(bus, [motor_listener, printer_listener, logger_listener])

# 30 saniye izle
print("CAN bus izleniyor... (30 saniye)")
time.sleep(30)

notifier.stop()
bus.shutdown()

08 J1939 ve DBC

J1939 ağır taşıt CAN protokolüdür; DBC dosyaları CAN sinyal tanımlamalarını içerir — ikisini birleştiren cantools, profesyonel CAN analiz altyapısının temelini oluşturur.

SAE J1939 nedir?

J1939, SAE International tarafından tanımlanan, ağır ticari araçlarda (kamyon, traktör, iş makinesi) kullanılan üst katman CAN protokolüdür. 29-bit Extended CAN ID kullanır ve ID içinde PGN, kaynak adresi ve hedef adres bilgisi kodlanmıştır.

  J1939 29-bit Extended CAN ID yapısı:
  ─────────────────────────────────────────────────────────
  Bit 28:    Priority (3 bit, 0=en yüksek)
  Bit 25:    Reserved
  Bit 24:    Data Page
  Bit 23–16: PF (PDU Format)  → PGN'nin parçası
  Bit 15–8:  PS (PDU Specific) / Destination Address
  Bit 7–0:   SA (Source Address)

  Örnek: 18 FF 00 E0  (hex, 29-bit)
         Priority=6, PGN=0xFF00, SA=0xE0

PGN ve SPN kavramları

KavramAçıklamaÖrnek
PGN (Parameter Group Number)Mesaj türünü tanımlar; hangi parametrelerin bir arada gönderildiğini belirtirPGN 61444 = Engine Torque/Speed
SPN (Suspect Parameter Number)Tek bir sinyal — PGN içindeki bit konumu, ölçek, offset tanımlanırSPN 190 = Engine Speed (0.125 RPM/bit)
SA (Source Address)Mesajı gönderen ECU'nun ağ adresi (0–253)0x00 = Engine, 0x11 = Dashboard

Linux kernel J1939 desteği

j1939_socket.c●
#include <linux/can.h>
#include <linux/can/j1939.h>

int sock = socket(PF_CAN, SOCK_DGRAM, CAN_J1939);

struct sockaddr_can addr = {
    .can_family       = AF_CAN,
    .can_ifindex      = ifr.ifr_ifindex,
    .can_addr.j1939 = {
        .name   = J1939_NO_NAME,
        .pgn    = 0xF004,          /* PGN 61444: Engine T/S */
        .addr   = 0x00,            /* SA: Engine ECU */
    },
};

bind(sock, (struct sockaddr *)&addr, sizeof(addr));

uint8_t payload[8];
ssize_t n = recv(sock, payload, sizeof(payload), 0);
/* n = 8 → PGN 61444 verisi: torque, speed, throttle */

DBC dosyası nedir?

DBC (Database CAN), Vector tarafından geliştirilen açık metin formatıdır. CAN mesajlarını ve içerdikleri sinyalleri tanımlar: hangi bit konumunda, hangi ölçek ve offset ile, hangi birimi kullandığını belirtir. Tüm büyük OEM'ler araç CAN ağlarını DBC ile belgeler.

dbc_decode.py●
import cantools
import can

# DBC dosyasını yükle
db = cantools.database.load_file("example.dbc")

# Mevcut mesajları listele
for msg in db.messages:
    print(f"  {msg.name}: ID=0x{msg.frame_id:03X}, {len(msg.signals)} sinyal")

# Ham byte dizisini decode et
raw_data = bytes([0x80, 0x0C, 0x5A, 0x7D, 0x00, 0x00, 0x00, 0x00])
decoded = db.decode_message(0x1F4, raw_data)
# → {'EngineSpeed': 1600.0, 'EngineTemp': 50, 'ThrottlePos': 49.02}

# SocketCAN ile gerçek zamanlı decode
bus = can.Bus(interface='socketcan', channel='vcan0')

while True:
    msg = bus.recv(timeout=1.0)
    if msg is None:
        continue
    try:
        decoded = db.decode_message(msg.arbitration_id, msg.data)
        print(f"[0x{msg.arbitration_id:03X}] {decoded}")
    except KeyError:
        # DBC'de tanımlanmamış ID
        pass
