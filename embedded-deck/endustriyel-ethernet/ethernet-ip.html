00 EtherNet/IP ve CIP nedir — ODVA ve tarihçe

EtherNet/IP (Ethernet Industrial Protocol), CIP'i (Common Industrial Protocol) standart Ethernet üzerinde taşıyan endüstriyel protokoldür. 2000 yılında Rockwell Automation ve ODVA (Open DeviceNet Vendors Association) tarafından geliştirilmiş; bugün dünyada en yaygın kullanılan endüstriyel Ethernet protokollerinden biridir.

CIP ailesi

          CIP (Common Industrial Protocol)
               │
    ┌──────────┼──────────────┐
    │          │              │
DeviceNet   ControlNet   EtherNet/IP
(CAN üzeri) (koaksiyel)  (TCP/UDP/IP)
 1994          1995          2000

CIP, nesne modeli, mesajlaşma mekanizmaları ve cihaz profilleri dahil tüm protokol davranışını tanımlar; alt taşıma katmanı değişebilir. DeviceNet CAN üzerinde, ControlNet koaksiyel kablo üzerinde, EtherNet/IP ise standart TCP/IP üzerinde çalışır.

EtherNet/IP temel özellikleri

Taşıma katmanıExplicit messaging için TCP port 44818, Implicit (I/O) messaging için UDP port 2222. Standart IP yönlendirme üzerinden çalışır.
Encapsulation protokolüEtherNet/IP, CIP mesajlarını "Encapsulation" protokolüyle TCP/UDP'ye sarar. Her Encapsulation paketi bir komut kodu, uzunluk ve oturum tanımlayıcı içerir.
ODVA standartlarıCIP Spesifikasyonu vol. 1-3: genel servisler, EtherNet/IP özel adaptasyonu ve cihaz profilleri (Drive, Motor Overload, Generic…).
Conformance testODVA uyum testi zorunludur. "EtherNet/IP" logosunu taşıyan cihazlar birbirleriyle uyumlu çalışır.
NOT

Rockwell Automation (Allen-Bradley) terminolojisinde "tag" kavramı, CIP Assembly nesnelerini soyutlar. CompactLogix/ControlLogix PLC'lerde program ve controller tag'leri doğrudan CIP path ile erişilebilir.

01 CIP nesne modeli — Identity, Assembly, Connection Manager

CIP'in temel soyutlama birimi "nesne"dir. Her cihaz zorunlu ve opsiyonel nesnelerden oluşan bir nesne modelini uygular. Nesneler sınıf (Class), örnek (Instance) ve özellik (Attribute) üçlüsüyle tanımlanır.

Temel CIP nesneleri

SınıfClass IDZorunlu mu?Açıklama
Identity Object0x01EvetVendor ID, Device Type, Product Code, Major/Minor revision, Serial Number, Product Name
Message Router0x02EvetGelen CIP mesajlarını ilgili nesneye yönlendirir
Connection Manager0x06EvetI/O ve explicit bağlantıların kurulması ve yönetimi
Assembly Object0x04Profil bağımlıBirden fazla attribute'i tek veri bloğunda birleştirir; I/O verisi için kullanılır
Parameter Object0x0FHayırCihaz parametrelerini yapılandırılmış şekilde sunar
TCP/IP Interface0xF5EvetIP adresi, subnet, ağ geçidi, hostname konfigürasyonu
Ethernet Link0xF6EvetMAC adresi, hız/duplex, arayüz istatistikleri
Port Object0xF4EvetCIP port tanımı (EtherNet/IP için port 2)

CIP path yapısı

Her CIP mesajı bir hedef path içerir. Path, hangi nesne/attribute'e erişileceğini belirtir:

Logical Segment Örnekleri:
┌─────────────────────────────────────────────────────┐
│  Class ID 0x01, Instance 1, Attribute 7             │
│  → Identity Object, örnek 1, "Product Name"         │
│  Path: 0x20 0x01 0x24 0x01 0x30 0x07               │
│        (Cls) (01)  (Ins) (01)  (Atr) (07)           │
│                                                     │
│  Class 0x04, Instance 0x64, Attribute 3             │
│  → Assembly Instance 100, veri attribute            │
│  Path: 0x20 0x04 0x24 0x64 0x30 0x03               │
└─────────────────────────────────────────────────────┘

Rockwell Tag (Symbolic Segment):
  Path: 0x91 0x0A "MotorSpeed\0"  (uzunluk=10)

Vendor ID ve Device Type

Vendor IDODVA tarafından atanan üretici kimliği. Rockwell: 1, Omron: 47, Schneider: 374, Allen-Bradley: 1.
Device TypeCihaz profili: 0x02 = AC Drive, 0x07 = General Purpose I/O, 0x0C = Communications Adapter, 0x22 = Safety Discrete I/O.
Product CodeÜretici tanımlı ürün kodu. Aynı vendor içinde benzersiz.

02 Implicit messaging — I/O bağlantısı ve Class 1

Implicit messaging (I/O messaging), cyclic gerçek zamanlı veri aktarımı için kullanılır. UDP üzerinden peer-to-peer çalışır; her pakette CIP header yoktur — sadece ham proses verisi taşınır. Bu nedenle "implicit" denir: ne gönderildiği önceden bilinir.

Class 1 bağlantı parametreleri

RPI (Requested Packet Interval)Cyclic güncelleme periyodu (mikrosaniye). Örn: 10000 = 10 ms. Gönderici bu hızda paket üretir.
O→T (Originator to Target)Bağlantıyı başlatan tarafın hedef cihaza gönderdiği veri yönü. PLC'den drive'a çıkış verisi.
T→O (Target to Originator)Hedef cihazdan başlatana dönen veri yönü. Drive'dan PLC'ye giriş verisi (gerçek akım, frekans, hata kodu).
Connection PointAssembly nesnesi örnek numarası. 0x64 (100): output assembly, 0x65 (101): input assembly, 0x66 (102): configuration assembly.
Exclusive OwnerBağlantı türü: Exclusive Owner (tek kontrol edici), Input Only (sadece oku), Listen Only (izleme).

ForwardOpen mesajı

İstemci (PLC)                          Sunucu (Drive)
    │                                       │
    │── ForwardOpen Request ───────────────►│
    │   Priority&TimeTick, TimeoutTicks     │
    │   O→T Network Conn. Params:           │
    │     Size: 4 bytes (O→T data)         │
    │     RPI: 10000 µs                    │
    │     Type: Point-to-Point, UDP        │
    │   T→O Network Conn. Params:           │
    │     Size: 8 bytes (T→O data)         │
    │     RPI: 10000 µs                    │
    │   Connection Path:                   │
    │     20 04 24 64  (Assembly 0x64)     │
    │     20 04 24 65  (Assembly 0x65)     │
    │                                       │
    │◄─ ForwardOpen Response ───────────────│
    │   O→T Conn ID: 0x12345678            │
    │   T→O Conn ID: 0x87654321            │
    │   O→T API: 10000 µs                  │
    │                                       │
    │══ UDP Cyclic I/O (10 ms periyot) ════►│
    │◄═ UDP Cyclic I/O (10 ms periyot) ════ │

Consumed / Produced tag (Rockwell)

Allen-Bradley ControlLogix/CompactLogix PLC'lerde tag bazlı I/O bağlantıları kullanılır. Bir tag "produced" (üretilen) veya "consumed" (tüketilen) olarak işaretlenebilir ve diğer PLC'lerle paylaşılabilir.

03 Explicit messaging — Class 3 ve UCMM

Explicit messaging, parametre okuma/yazma ve cihaz konfigürasyonu için kullanılır. TCP üzerinden istek-yanıt modeliyle çalışır; her mesaj hangi nesne/attribute'e erişildiğini açıkça belirtir. Bu nedenle "explicit" denir.

Explicit messaging türleri

UCMM (Unconnected Message Manager)Bağlantı kurmadan tek seferlik mesaj gönderme. Basit parametre okuma için uygundur. Ek yük fazladır.
Class 3 (Connected Explicit)Önce TCP bağlantısı kurulur, ardından bu bağlantı üzerinden mesajlar gönderilir. Çok sayıda istek yapılacaksa daha verimlidir.
Large Forward OpenStandart ForwardOpen maksimum 508 byte veri destekler. Large Forward Open ile 65535 byte'a kadar çıkılır.

Temel CIP servis kodları

Servis KoduAdAçıklama
0x01Get_Attributes_AllTüm attribute'leri tek seferde oku
0x02Set_Attributes_AllTüm attribute'leri tek seferde yaz
0x0EGet_Attribute_SingleTek bir attribute oku
0x10Set_Attribute_SingleTek bir attribute yaz
0x4CRead_TagRockwell'e özel: sembolik tag oku
0x4DWrite_TagRockwell'e özel: sembolik tag yaz
0x4ERead_Tag_FragmentedBüyük array'leri parçalar halinde oku
0x4FWrite_Tag_FragmentedBüyük array'leri parçalar halinde yaz
0x55Read_Modify_Write_TagAtomik okuma-değiştirme-yazma

06 pycomm3 Python kütüphanesi

pycomm3, Allen-Bradley CompactLogix ve ControlLogix PLC'lerle iletişim için en popüler Python kütüphanesidir. Logix5000 tag protokolünü uygular; sembolik tag adlarıyla doğrudan okuma/yazma yapılır.

pip install pycomm3

Temel bağlantı ve tag okuma

from pycomm3 import LogixDriver

# CompactLogix/ControlLogix PLC bağlantısı
# IP adresi, slot numarası (backplane slot)
with LogixDriver('192.168.1.1/0') as plc:
    # Tek tag oku
    tag = plc.read('Motor.Speed')
    print(f"Motor Hızı: {tag.value} RPM")
    print(f"Veri tipi:  {tag.type}")
    print(f"Hata:       {tag.error}")

    # Birden fazla tag aynı anda oku (verimli)
    tags = plc.read('Motor.Speed', 'Motor.Running', 'Conveyor.Position')
    for t in tags:
        print(f"{t.tag}: {t.value} ({t.type})")

Tag yazma

from pycomm3 import LogixDriver, DINT, REAL, BOOL, STRING

with LogixDriver('192.168.1.1/0') as plc:
    # Tek tag yaz
    result = plc.write(('Motor.SpeedRef', 1500.0))
    print(f"Yazma {'başarılı' if result.error is None else 'başarısız'}")

    # Çoklu yazma (tek seferde gönderilir — verimli)
    results = plc.write(
        ('Motor.SpeedRef',   1500.0),   # REAL
        ('Motor.RunCmd',     True),     # BOOL
        ('Conveyor.MaxPos',  5000),     # DINT
        ('System.Message',   "Calisiyor"),  # STRING
    )
    for r in results:
        print(f"{r.tag}: {r.error or 'OK'}")

07 Pratik: Allen-Bradley PLC'den tag okuma ve CSV loglama

Bu bölümde Allen-Bradley CompactLogix PLC'den belirli tag'leri periyodik olarak okuyup CSV dosyasına kaydeden eksiksiz bir izleme uygulaması geliştirilir.

Sistem mimarisi

CompactLogix PLC
192.168.1.1/0
    │
    │ EtherNet/IP (TCP 44818)
    │ Explicit Messaging (CIP Read_Tag)
    │
Linux PC / Raspberry Pi
pycomm3 Python uygulaması
    │
    ├── Ekran çıktısı (real-time)
    └── CSV dosyası (/var/log/plc_data.csv)

#!/usr/bin/env python3
"""
plc_logger.py — Allen-Bradley PLC tag izleme ve CSV loglama
Gereksinim: pip install pycomm3
Kullanım:   python3 plc_logger.py --ip 192.168.1.1 --slot 0 --interval 1.0
"""
import csv
import time
import signal
import logging
import argparse
from datetime import datetime
from pathlib import Path
from pycomm3 import LogixDriver, RequestError, CommError

logging.basicConfig(
    level=logging.INFO,
    format="%(asctime)s %(levelname)s %(message)s"
)
logger = logging.getLogger(__name__)

# İzlenecek tag listesi
TAGS_TO_MONITOR = [
    "Motor1.Speed",          # REAL  — motor hızı (RPM)
    "Motor1.Current",        # REAL  — motor akımı (A)
    "Motor1.Running",        # BOOL  — çalışıyor mu?
    "Motor1.FaultCode",      # DINT  — hata kodu
    "Conveyor.Position",     # DINT  — konveyör pozisyonu (mm)
    "Conveyor.Speed",        # REAL  — konveyör hızı (m/min)
    "Tank.Level",            # REAL  — tank seviyesi (%)
    "Tank.Temperature",      # REAL  — sıcaklık (°C)
    "System.CycleCount",     # DINT  — üretim döngü sayacı
    "System.ProductionOK",   # BOOL  — ürün kabul/red
]

# Alarm eşikleri (tag_adı: (min, max))
ALARM_THRESHOLDS = {
    "Motor1.Current":  (0.0,  15.0),   # 15 A üzeri alarm
    "Tank.Temperature": (0.0, 85.0),   # 85 °C üzeri alarm
    "Tank.Level":      (10.0, 95.0),   # %10 altı veya %95 üzeri
}

running = True

def signal_handler(sig, frame):
    global running
    logger.info("Durdurma sinyali alındı...")
    running = False

def check_alarms(tag_name: str, value) -> str | None:
    """Değeri eşiklerle karşılaştır; alarm mesajı döndür veya None."""
    if tag_name not in ALARM_THRESHOLDS:
        return None
    lo, hi = ALARM_THRESHOLDS[tag_name]
    try:
        v = float(value)
        if v < lo:
            return f"DÜŞÜK ({v:.2f} < {lo})"
        if v > hi:
            return f"YÜKSEK ({v:.2f} > {hi})"
    except (TypeError, ValueError):
        pass
    return None

def read_all_tags(plc: LogixDriver) -> dict:
    """Tüm tag'leri oku; hatalıları None olarak döndür."""
    results = {}
    try:
        readings = plc.read(*TAGS_TO_MONITOR)
        # Tek tag olsa da liste döner (normalize)
        if not isinstance(readings, (list, tuple)):
            readings = [readings]
        for r in readings:
            results[r.tag] = r.value if r.error is None else None
            if r.error:
                logger.warning(f"Tag okuma hatası [{r.tag}]: {r.error}")
    except (RequestError, CommError) as e:
        logger.error(f"PLC iletişim hatası: {e}")
    return results

def setup_csv(filepath: Path) -> csv.DictWriter:
    """CSV dosyasını oluştur veya mevcut olanı aç."""
    fieldnames = ["timestamp"] + TAGS_TO_MONITOR + ["alarmlar"]
    file_exists = filepath.exists()
    f = open(filepath, "a", newline="", encoding="utf-8")
    writer = csv.DictWriter(f, fieldnames=fieldnames)
    if not file_exists:
        writer.writeheader()
        logger.info(f"CSV dosyası oluşturuldu: {filepath}")
    return writer, f

def main():
    parser = argparse.ArgumentParser(description="PLC Tag Logger")
    parser.add_argument("--ip",       default="192.168.1.1")
    parser.add_argument("--slot",     type=int, default=0)
    parser.add_argument("--interval", type=float, default=1.0,
                        help="Örnekleme periyodu (saniye)")
    parser.add_argument("--output",   default="/var/log/plc_data.csv")
    args = parser.parse_args()

    signal.signal(signal.SIGINT,  signal_handler)
    signal.signal(signal.SIGTERM, signal_handler)

    csv_path = Path(args.output)
    csv_path.parent.mkdir(parents=True, exist_ok=True)
    writer, csv_file = setup_csv(csv_path)

    plc_addr = f"{args.ip}/{args.slot}"
    logger.info(f"PLC'ye bağlanıyor: {plc_addr}")
    logger.info(f"Örnekleme aralığı: {args.interval} s")
    logger.info(f"CSV çıktı: {csv_path}")
    logger.info(f"İzlenen tag sayısı: {len(TAGS_TO_MONITOR)}")

    reconnect_delay = 5.0
    plc = None

    try:
        while running:
            loop_start = time.monotonic()

            # PLC bağlantısı yok ise kur
            if plc is None:
                try:
                    plc = LogixDriver(plc_addr)
                    plc.open()
                    info = plc.info
                    logger.info(f"Bağlantı kuruldu: "
                                f"{info.get('product_name', 'N/A')} "
                                f"Rev:{info.get('revision', 'N/A')}")
                except (CommError, Exception) as e:
                    logger.error(f"Bağlantı kurulamadı: {e}")
                    time.sleep(reconnect_delay)
                    continue

            # Tag'leri oku
            data = read_all_tags(plc)

            if not data:
                logger.warning("Veri alınamadı, yeniden bağlanılıyor...")
                try:
                    plc.close()
                except Exception:
                    pass
                plc = None
                time.sleep(reconnect_delay)
                continue

            # Alarm kontrolü
            alarms = []
            for tag_name, value in data.items():
                if value is not None:
                    alarm = check_alarms(tag_name, value)
                    if alarm:
                        alarms.append(f"{tag_name}: {alarm}")
                        logger.warning(f"ALARM — {tag_name}: {alarm}")

            # Ekran çıktısı
            now = datetime.now()
            print(f"\n{'='*60}")
            print(f"  {now:%Y-%m-%d %H:%M:%S.%f}"[:28])
            print(f"{'='*60}")
            for tag_name in TAGS_TO_MONITOR:
                v = data.get(tag_name)
                val_str = f"{v:.3f}" if isinstance(v, float) else str(v)
                alarm_str = ""
                if any(tag_name in a for a in alarms):
                    alarm_str = " ⚠ ALARM"
                print(f"  {tag_name:<28} = {val_str}{alarm_str}")
            if alarms:
                print(f"\n  AKTIF ALARMLAR ({len(alarms)}):")
                for a in alarms:
                    print(f"    ! {a}")

            # CSV'ye yaz
            row = {"timestamp": now.isoformat(), "alarmlar": "|".join(alarms)}
            row.update({t: data.get(t, "") for t in TAGS_TO_MONITOR})
            writer.writerow(row)
            csv_file.flush()

            # Periyot bekle
            elapsed = time.monotonic() - loop_start
            wait = args.interval - elapsed
            if wait > 0:
                time.sleep(wait)
            else:
                logger.warning(f"Döngü süresi aşıldı: {elapsed:.3f} s")

    finally:
        logger.info(f"Toplam {csv_path} KB kayıt yazıldı")
        csv_file.close()
        if plc is not None:
            try:
                plc.close()
            except Exception:
                pass
        logger.info("Logger durduruldu.")

if __name__ == "__main__":
    main()
